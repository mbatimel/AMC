# ВИ-Портал

Корпоративный веб-портал для ООО ПО «ВОЛЖСКИЙ ИНСТРУМЕНТ»: публичный каталог,
электронные заказы, личный кабинет клиента, интеграция с 1С, публикация товаров
на маркетплейсах и управление складскими остатками.


## Структура проекта

```text
.
├── back/
│   ├── helper/
│   ├── integrations/
│   ├── migrations/
│   ├── orders/
│   ├── platforms/
│   ├── products/
│   ├── users/
│   └── warehouses/
└── front/
```

## Backend-сервисы

| Папка | Модуль ТЗ | Назначение |
| --- | --- | --- |
| `back/products` | M-01 | Каталог продукции: товары, категории, характеристики, цены, поиск, прайс-листы. |
| `back/orders` | M-02 | Электронные заказы: корзина, оформление заказа, статусы, уведомления менеджеру и клиенту. |
| `back/integrations` | M-03 | Интеграция с 1С: синхронизация номенклатуры, цен, остатков, клиентов, заказов и счетов. |
| `back/helper` | M-04 | Интеллектуальный помощник: подбор инструмента по запросу клиента и связь с каталогом. |
| `back/users` | M-05 | Пользователи и личный кабинет: регистрация, роли, клиенты, индивидуальные условия, история заказов. |
| `back/platforms` | M-07 | Маркетплейсы: публикация и обновление товаров на Ozon, Wildberries и Яндекс Маркет. |
| `back/warehouses` | M-08 | Складские остатки: остатки по складам, пороги, блокировка заказа при отсутствии товара, уведомления. |
| `back/migrations` | - | Общие миграции базы данных. Это инфраструктурная папка, не отдельный микросервис. |

## Рекомендуемые недостающие сервисы

В ТЗ также есть модули, под которые пока нет отдельных папок:

| Рекомендуемая папка | Модуль ТЗ | Назначение |
| --- | --- | --- |
| `back/billing` | M-06 | Генерация счетов, накладных, УПД и PDF-документов по заказам. |
| `back/forecasting` | M-09 | Прогнозирование спроса, аналитика продаж и рекомендации по пополнению склада. |

Модуль M-09 в ТЗ отмечен как опциональный и включается в объем разработки после
подтверждения заказчиком.

## Планируемый стек

Согласно ТЗ backend может быть реализован на Python FastAPI или Go.

Рекомендуемые компоненты:

- PostgreSQL 14+ для основного хранилища.
- Redis и очередь задач для фоновых операций и интеграций.
- S3-совместимое хранилище для фотографий товаров и документов.
- OpenSearch, Elasticsearch или PostgreSQL FTS для поиска по каталогу.
- Docker Compose или Kubernetes для развертывания.
